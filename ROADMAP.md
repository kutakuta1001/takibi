## フェーズと受け入れ条件

### Phase 0: スキャフォールド（計画 Task 1〜4）
起伏のある地形と川筋のある世界を、一人称視点で歩き回れる。
- 受け入れ: クリックで開始→PointerLock→WASD歩行、地形テスト（Vitest）グリーン、build グリーン

### Phase 1: 森の世界（計画 Task 5〜8）
森・昼夜・環境音がそろい、「そこに居る」感覚が成立する。
- 受け入れ: 木々の中を歩ける（60fps近辺）、昼夜が移ろう、風・川（距離連動）・鳥/虫の音が鳴る

### Phase 2: 体験コア（計画 Task 9〜14）— 本作の価値の中心
伐採→薪→焚き火が育つ→水汲み→コーヒー→座って飲む、の連鎖が完成する。
- 受け入れ: 計画 Task 14 のチェックリスト全項目、GameState テスト全グリーン

### Phase 2.5: ビジュアルリアリティ強化（計画: `docs/superpowers/plans/2026-07-07-visual-realism-plan.md`）
CEO フィードバック（2026-07-07「もう少しリアルな森・川に」）への対応。CC0 写真テクスチャ +
影・トーンマッピング・水面シェーダで実在感を上げる。Phase 2 完了後に着手。
- 受け入れ: 地面・樹皮・岩・水面に実写質感、木漏れ日の影、60fps 近辺を維持、CEO が「リアルになった」と感じること

### Phase 3: 磨き（計画未作成。Phase 2.5 完了後に writing-plans で作成）
- SnowTheme（雪山切替）、蛍、タイトル仕上げ、音響強化（PannerNode・リバーブ）、モバイル検討
- 受け入れ: タイトルで森林/雪山を選択でき、雪山で降雪・雪面・静けさの音場になる

## バックログ（発案記録・採否未定）

### Google マップの写真活用（2026-07-07 CEO 発案）

発案趣旨: 実在の風景写真を利用すればよりリアルになるのではないか。採用は必須ではない。

技術・ライセンス評価（記録時点）:
- Google マップ / ストリートビューの画像を抽出してテクスチャ等の素材に使うことは Google の
  利用規約で禁止されている（画像からの派生物作成不可）。写真の直接流用は不採用とする
- 正規ルートは Google Maps Platform の Photorealistic 3D Tiles API。three.js からも利用できるが、
  有料・オンライン必須・APIキー管理が必要で、空撮フォトグラメトリのため地上の一人称視点では
  近景が溶けたように見える。本作の「足元の実在感」には不向き
- 同じ狙いを満たす代替案（こちらが有望）:
  1. Poly Haven の実写森林 HDRI を空・環境光に使う（空の実在感が大きく向上。昼夜サイクルとの
     両立方法の設計が必要）
  2. フォトグラメトリ由来の CC0 3Dモデル（岩・切り株・倒木）を点在させる（設計書C案の具体化）
- 採用可能性のある形: 「実在の山並みを遠景に眺める」演出、開始画面で実在の地図から
  キャンプ地を選ぶ演出など、限定的な利用

判断タイミング: Phase 3 計画作成時に再評価する。
