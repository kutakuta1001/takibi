# 設計書: Takibi（仮称）— ブラウザで味わうキャンプ体験

日付: 2026-07-07
状態: CEO承認済み（舞台・表現方式・対象環境・トーンの4点は対話で確定）
作成: Claude Fable 5（引き継ぎ先: Opus 4.8 / Sonnet 5）

## 1. コンセプト

ウェブブラウザを開くと森の中のキャンプサイトに立っており、歩き、木を切って薪にし、
川から水を汲み、焚き火でコーヒーを淹れて飲む——「キャンプをした実感」を音と情景で
味わう没入型体験アプリ。建築・クラフトを楽しむゲームではない。ノルマ・ゲームオーバー・
数値パラメータ管理は置かない。行動が軽く連鎖する達成感（薪→火が育つ→コーヒー）だけを残す。

確定した方針（CEO回答）:

| 論点 | 決定 |
|---|---|
| 舞台 | 森林から作る。雪山はテーマ差し替えで後続対応（Theme 設計を最初から入れる） |
| 表現 | 3D一人称（Three.js）。歩いて川へ行き、木に近づいて斧を振る |
| 対象 | PCブラウザ優先（キーボード+マウス）。モバイルは後続フェーズ |
| トーン | 癒し + 軽い連鎖。サバイバル要素なし |

## 2. 技術選定

- **Vanilla Three.js + TypeScript + Vite**（採用）
  - 理由: 依存最小・Sonnet 5 / Opus 4.8 の知識が最も安定・この体験規模に十分
  - 不採用: React Three Fiber（React層がデバッグと引き継ぎの前提知識を増やすが、UI最小の本作では恩恵が薄い）、Babylon.js 等エンジン（機能過剰）
- **物理エンジンなし**: 地形の高さ関数 `heightAt(x, z)` にカメラを追従させる方式で十分
- **ジオメトリは手続き生成、質感は CC0 写真テクスチャ**（2026-07-07 CEO 承認によりルール変更）:
  木・地形・焚き火の形状は手続き生成を維持しつつ、Poly Haven / ambientCG の CC0 テクスチャ
  （color / normal / roughness、1K〜2K）をリポジトリ内 `public/textures/` に保存して使用する。
  出典は `public/textures/ATTRIBUTION.md` に記録する。GLTF 等の外部3Dモデル導入（C案）は
  テクスチャ強化の結果を CEO が評価してから判断する。
  変更前の方針（完全アセットレス）は Phase 1 まで適用されていた
- **音は Web Audio API による手続き合成を第一候補**: 風=フィルタ付きノイズ、川=変調ノイズ+
  バンドパス、焚き火=ランダムパルスのクラックル、鳥=FM合成チャープ。品質が不足する音のみ
  CC0素材（freesound等）へ差し替え可（差し替えやすいよう AudioEngine にソース抽象を置く）
- **テスト**: Vitest。ロジック層（GameState・火の燃料・インタラクション状態遷移）を単体テスト。
  ビジュアル・音は手動確認
- **ポストプロセスは使わない**: フォグ・色設計・ライティングで雰囲気を作る。60fps目標（ミッドレンジPC）

## 3. アーキテクチャ

```
src/
  main.ts                 エントリ。Engine・World・Systems・UI の結線のみ
  core/
    Engine.ts             renderer / scene / camera / clock / resize / メインループ
    Input.ts              キー状態・PointerLock 管理
    PlayerController.ts   WASD移動・視点・地形高さへのクランプ・歩行ボブ（控えめ）
  world/
    Terrain.ts            SimplexNoise 起伏地形。heightAt(x,z) を公開。川筋は高さを掘り下げる
    River.ts              水面平面（シェーダで流れ表現）・水汲み可能ゾーン定義
    Forest.ts             背景の木は InstancedMesh 数百本。伐採可能な木はキャンプ場近くに個別メッシュで5〜8本
    Sky.ts                昼夜サイクル（実時間約10分/周・設定可）。空色グラデーション・太陽/月光・夜の星
    Particles.ts          焚き火の火の粉・夜の蛍。（雪はSnowTheme側で使用）
  theme/
    Theme.ts              interface Theme { sky, fog, ground, ambientSoundSet, particles, treeParams }
    ForestTheme.ts        本フェーズで実装
    SnowTheme.ts          Phase 3。色・フォグ・地面・音・降雪パーティクルの差し替えのみで雪山化
  systems/
    GameState.ts          純TSの中央状態+イベントエミッタ（描画非依存・単体テスト対象）
    Interaction.ts        カメラ中心レイキャスト（射程約3m）。Interactable インターフェース
    Chopping.ts           木: Eキー数回で伐倒アニメ→薪ピックアップ化
    Fire.ts               焚き火: 薪をくべる→fuel増加→炎スケール・光量・音量が連動。fuelは緩やかに減衰
    Water.ts              川ゾーンでケトルに水を汲む
    Cooking.ts            水入りケトルを焚き火へ→湯気→コーヒー完成→座って飲む（カメラ降下・すする音）
  audio/
    AudioEngine.ts        AudioContext管理・初回操作でのアンロック・マスターゲイン・距離減衰
    synths.ts             手続き合成音源（風・川・焚き火・鳥・虫・SFX: 斧/水汲み/すする音）
  ui/
    Title.ts              タイトル画面（クリックで開始 = PointerLock取得とAudioContextアンロックを兼ねる）
    HUD.ts                中央ドット・文脈プロンプト（「Eで木を切る」等）・最小の所持品トレイ（DOMオーバーレイ）
```

設計原則:

- GameState は Three.js に依存しない純ロジック。systems が GameState を書き換え、
  world/audio がイベント購読で反応する一方向データフロー
- Interactable は `prompt(state)` / `canInteract(state)` / `interact(state)` の3メソッド。
  新しい体験（例: テント設営）は Interactable 追加のみで拡張可能
- Theme は起動時に注入。ForestTheme/SnowTheme の切り替えは main.ts の1箇所

## 4. 体験ループ（コア仕様）

1. タイトル画面「クリックではじめる」→ フェードインで森へ。環境音が立ち上がる
2. 一人称で歩き回る（WASD+マウス、Shiftは不要=走らない世界観）
3. 伐採可能な木に近づく → プロンプト「Eで木を切る」→ Eを3〜5回（斧音・木の揺れ）→ 伐倒 → 薪3本に変化
4. 薪を拾う（E）→ 所持品トレイに薪カウント
5. 焚き火サイトで E → 薪をくべる → 火が育つ（炎の大きさ・パチパチ音・光の届く範囲が fuel に連動。
   火が小さいと夜が暗く心細い、が演出の核）
6. 川辺で E → ケトルに水を汲む（水音）
7. 焚き火の前で E → ケトルを火にかける → 数十秒で湯気 → 「コーヒーができた」
8. E で座って飲む → カメラが座り高さへ・視点がゆっくり炎へ → すする音・湯気 → 立ち上がる
9. 昼夜サイクルが背景で進行。夜は星空・虫の声・焚き火の存在感が最大化

エラー系（静かに誘導する）: 薪なしで焚き火にE→「薪がない。木を切ろう」、空ケトルで火にE→
「先に川で水を汲もう」のようにプロンプト文言で次の行動を示す。失敗音・警告UIは出さない。

## 5. 音響設計（体験の半分は音）

- レイヤー構成: ベース風音（常時・昼夜で強度変化）/ 鳥（昼）/ 虫（夜）/ 川（距離減衰・
  近づくほど大きく=PannerNode または距離ゲイン）/ 焚き火クラックル（fuel連動・距離減衰）
- SFX: 斧ヒット・伐倒・薪拾い・水汲み・ケトル設置・コーヒーすすり
- すべて synths.ts の関数として実装し、AudioEngine 経由で再生。CC0差し替え時は
  同名インターフェースでファイル再生に置換できる構造にする

## 6. テスト戦略

- Vitest: GameState の遷移（薪カウント・ケトル状態機械 empty→filled→onFire→ready→empty・
  fire fuel の増減）・Interactable の canInteract 条件
- 手動確認チェックリスト（各フェーズの受け入れ条件として計画書に記載）
- `npm run dev` / `npm run build` / `npm run test` の3コマンドを常にグリーンに保つ

## 7. フェーズ構成（詳細はROADMAP.md）

- Phase 0: スキャフォールド — 歩ける地形・空・フォグ・PointerLock（動く最小体験）
- Phase 1: 森の世界 — 木・川・昼夜サイクル・環境音レイヤー
- Phase 2: 体験コア — 伐採・焚き火・水汲み・コーヒー（本作の価値の中心）
- Phase 3: 磨き — SnowTheme・タイトル画面仕上げ・音響強化・モバイル対応検討

## 8. スコープ外（YAGNI）

- 建築・クラフトメニュー・テックツリー / セーブデータ（体験は毎回まっさらでよい） /
  マルチプレイ / 動物AI / 体温・空腹などのサバイバルパラメータ / VR対応
