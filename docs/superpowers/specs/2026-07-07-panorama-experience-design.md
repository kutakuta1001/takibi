# 設計書 v2: Takibi — 実写360°パノラマ・キャンプ体験（方向転換）

日付: 2026-07-07
状態: CEO承認済み。v1（3D歩行版、`2026-07-07-camp-experience-design.md`）を置き換える。
v1 実装は git ブランチ `archive/3d-walkable` に保全済み。

## 1. 方向転換の経緯と方針

CEO フィードバック（v1 の Phase 2.5 + V7 完了後）:
「音声ややりたいことの体験は素晴らしい。没入感が一番大事なのでリアルな写真を出すことを
優先し、動き回ることはなしでも構わない」

これを受けた確定方針:

| 論点 | 決定 |
|---|---|
| 映像 | CC0 の実写360°パノラマ写真（Poly Haven 等の equirectangular）を球体内面に投影 |
| 視点 | 固定地点。マウスドラッグで見回しのみ（慣性付き）。WASD移動・PointerLock は廃止 |
| 場所移動 | 「スポット」間のクロスフェード遷移（例:「川辺へ水を汲みに行く」）。歩行の代替 |
| 体験連鎖 | v1 と同一（伐採→薪→焚き火→水汲み→コーヒー→座って飲む）。ホットスポット化して維持 |
| 音 | v1 の手続き合成エンジンを無改修で継続（CEO 高評価点） |
| ロジック | GameState・テストを無改修で継続 |
| 昼夜 | 実写は時刻固定のため「夕暮れ⇔夜」の色調グレーディングループ + 星の合成に置換 |
| 雪山対応 | パノラマ写真セット + 音セットの差し替え（v1 の Theme 思想を継承。実現は容易になる） |

## 2. アーキテクチャ（v1 からの差分）

維持（無改修）: `systems/GameState.ts`（+テスト）、`audio/AudioEngine.ts`、`audio/synths.ts`、
`ui/HUD.ts`、`ui/Title.ts`（クリック開始 = AudioContext アンロック）

削除: `core/PlayerController.ts`、`world/Terrain.ts`、`world/Forest.ts`、`world/Sky.ts`、
`world/River.ts`、地形テスト、`public/textures/` のうち前景3Dで使わないもの（bark/grass/ground。
rock は焚き火の石で継続使用）

新規:

```
src/pano/
  PanoScene.ts     equirectangular JPG を半径50の反転球へ。スポットごとに1シーン
  LookControls.ts  マウスドラッグ見回し（慣性・pitch clamp ±80度・感度控えめ）
  SpotManager.ts   スポット定義・クロスフェード遷移（1.5秒）・スポット別の環境音ミックス
  Grading.ts       夕⇔夜の色調ループ（パノラマ球のシェーダで露出/色温度/彩度を補間）+ 星オーバーレイ
  Hotspot.ts       パノラマ空間に固定した不可視コライダー + 視線/クリック判定（v1 Interactable 継承）
src/foreground/
  Fire.ts          v1 の焚き火を移植（炎・光・火の粉・クラックル音）。environment に PMREM(パノラマ) を
                   使い実写と馴染ませる。接地は暗いラジアルグラデのデカールで擬似影
  Cooking.ts       v1 を移植。座って飲む演出は「視点がゆっくり下がり炎に向く」カメラ演出に置換
                   （移動が無いため v1 で見送った自動視点誘導が安全に実装できる）
  Chopping.ts      斧ビューモデル + 伐採ホットスポット（パノラマ内の実際の木の位置に配置）
```

スポット構成（初期2つ）:
- `campsite`（メイン・夕暮れの森）: 焚き火・ケトル・薪くべ・伐採ホットスポット・座って飲む
- `riverside`（川辺）: 水汲みホットスポット。川音が主役になるミックス

## 3. アセット方針

- パノラマ: Poly Haven の CC0 HDRI を**トーンマップ済み 4K JPG**で取得（EXR は使わない）。
  夕方〜ゴールデンアワーの森 1 枚 + 川辺 1 枚。`public/panos/` に保存、
  出典は `public/panos/ATTRIBUTION.md` に記録。合計 30MB 以下
- 選定基準: 焚き火を置ける開けた地面がある / 視線の高さが人の立ち位置 / 川辺は水面が近い。
  実装時に Poly Haven を実際に閲覧して選定し、選定理由を報告する
- 炎の表現: (a) v1 の3Dパーティクル炎の移植 と (b) CC0 実写炎動画のビルボード合成 を
  実装時に両方試し、実写背景に馴染む方を採用（比較スクリーンショットを残す）

## 4. 昼夜の扱い

- ベース写真はゴールデンアワー。10分周期で「夕暮れ→夜→夕暮れ」をグレーディングでループ
  （露出低下・青方偏移・彩度低下。夜は星オーバーレイをフェードイン、焚き火の光の存在感が最大化）
- 鳥（夕）と虫（夜）の切替は既存の dayness 連動を流用（Grading が dayness を提供する）

## 5. テスト戦略

- GameState テスト（8件中5件）は無改修で維持。地形テスト3件は Terrain とともに削除
- 新規ユニットテスト: SpotManager の遷移状態機械、Grading の dayness 計算
- ビジュアル・音は playwright スクリーンショット自己確認（確立済み手法）+ CEO 実機確認
