# 設計書 v3: Takibi — 静かなサウンドノベル（選択肢駆動への転換）

日付: 2026-07-11
状態: CEO 設計承認済み（会話ベース）。設計書本文のレビュー待ち。
v2（`2026-07-07-panorama-experience-design.md`）のうち操作モデルと進行構造を置き換える。
パノラマ表示・手続き合成音・前景演出・写真選定原則など v2 の資産と思想は継続する。

## 1. 転換の経緯と方針

CEO フィードバック（Phase D 公開版確認後・2026-07-11）:
「音と画像はとてもいい。『E』を押して巻き戻したりなどのボタンは特に必要ないので削除。
文章が出てきて選択肢を選んでいくことでイベントが進んでいく、ゲームのような仕様にして、
もう少し行ける場所やキャンプのシーンを増やしたい」

これを受けた確定方針:

| 論点 | 決定 |
|---|---|
| 操作モデル | サウンドノベル型。画面下部のテキスト+選択肢で進行し、選ぶと行動が自動演出される（カメラが対象へゆっくり向き、既存の演出と音が流れる）。E キーと「Eで〜」プロンプトは全廃。クリック/タップのみで完結 |
| 分岐 | ゆるやかな分岐。選択で見るシーン・聞こえる音・描写が変わるが、必ず合流して良い一日で終わる。失敗・バッドエンドは存在しない |
| 構成軸 | 一日の流れ（午後→夕方→夜→朝）。一泊のキャンプを 15〜20 分で過ごし、静かに幕 |
| 時間帯表現 | 写真は場所ごとに 1 枚のまま。グレーディング + 音ミックス + StarMask で時間帯を表現。Grading の 10 分自動ループは廃止し、物語の進行が時間帯を進める |
| 雪山 | 朝のフィナーレ「山頂の一杯」として日循環に組み込む（E1 の体験を物語の結末に昇格） |
| 追加スポット | 湖畔・森の小道の 2 箇所（CC0 パノラマ写真の実在調査で最終確定） |
| 音 | 手続き合成エンジンを無改修で継続（synths.ts のアルゴリズム不変更） |
| ゲーム性の線引き | 選択はあるが試練はない。数値ノルマ・実績・カウンター・失敗演出は引き続き追加しない |

v2 の「体験の判断基準」は次の通り更新する:
「ゲームではなく体験」→「静かなサウンドノベル。選択はあるが試練はない」。
誘導をプロンプト文言で静かに行う思想（例「薪がない。木を切ろう」)は、シナリオ本文の文体として継承する。

## 2. 一日の流れとシナリオ構造

章 = 時間帯。各章はいくつかの「場面（ノード）」で構成し、
本文（静かな情景描写 1〜3 文）+ 選択肢（最大 3 つ程度）→ 行動演出 → 次の場面、で進む。

- 午後: 到着 → 薪を集める（森の小道へ拾いに行く / その場の木を切る）→ 川辺で水を汲む
- 夕方: 湖畔で夕日を見る / 焚き火を早めに起こす / 川辺で過ごす — 選択で見るシーンが変わる
- 夜: 焚き火・薪くべ・コーヒー抽出・星空。湖畔を選べば星の映り込み
- 朝: 霧の中で一杯 / 山頂へ登って一杯（雪山スポット・朝グレーディング）→ 静かに幕

構造のルール:

- 分岐は章の中で開き、章の終わりで必ず合流する（章をまたぐフラグ持ち越しは「描写の変化」まで。
  到達可否には使わない）
- 選択履歴は GameState 内の `Set<string>` に保持し、選択肢の表示条件と描写の変化にのみ使う。
  達成率・回想リストなどの可視化はしない
- 資源状態（薪・ケトル）は引き続き GameState が持ち、選択肢の表示条件に使う
  （例: 薪が無ければ「焚き火を起こす」は出さず「薪がない。木を切ろう」の流れへ導く）
- 幕が下りた後はタイトルに戻る。「もう一度過ごす」= 状態リセットで再演。選ばなかった
  シーンが次回の楽しみになる

## 3. 画面と操作

StoryPanel（新規・画面下部）:

- 半透明の帯に本文と選択肢ボタン。選択肢は実 button 要素とし Tab 到達性を確保する
- 選択すると選択肢が静かにフェードアウトし、演出が終わると次の場面の本文がフェードイン
- 演出中は選択肢を出さない。見回しは常に自由（演出の自動カメラパンはドラッグで中断できる）
- IdleWatcher の消灯対象外（「行動の入り口は迷いなく見える」原則。2026-07-11 CLAUDE.md 更新に準拠）

その他の UI:

- HotspotMarker: クリック機能を持たない「灯り」に転換。現在の選択肢が指す場所と連動して点灯し、
  どの方向の話かを静かに示す
- AreaTitle: 章タイトル（「夕暮れ」など）の表示に転用
- HUD: 中央プロンプト（「Eで〜」）は StoryPanel に置換。所持品トレイは雰囲気の小物として維持
- Help: 「ドラッグで見回す・選択肢で過ごす」に簡素化。H キー開閉は維持
- hints.ts: 役割を StoryEngine が吸収するため廃止（テスト 11 件も削除）

## 4. アーキテクチャ

新規（いずれも Three.js 非依存の純ロジック + TDD 対象）:

```
src/story/
  scenario.ts     シナリオデータ。ノードID・章（時間帯）・本文・選択肢（行き先/表示条件/演出指示）
  StoryEngine.ts  現在ノード管理・選択の適用・表示条件評価。GameState と同格の単体テスト対象
```

新規（演出・DOM）:

```
src/story/Direction.ts  演出ディスパッチャ。選択肢の演出指示を「カメラ自動パン + 既存演出関数 +
                        スポット遷移 + 時間帯遷移」へ変換する。main.ts の肥大化を防ぐため
                        story 配下に置き、main.ts は配線のみ担う
src/ui/StoryPanel.ts    画面下部のテキスト + 選択肢
```

変更:

- `pano/LookControls.ts`: 自動パン `glideTo(direction)` を追加（ドラッグで中断可能）
- `pano/Grading.ts`: 10 分自動ループを廃止し、時間帯プリセット（午後/夕/夜/朝)への
  滑らかな遷移に変更。朝プリセット（低彩度・霧感）を追加
- `systems/Interaction.ts`: レイキャスト + E/クリック仲介を撤去（Interactable インターフェース廃止）
- `foreground/Chopping.ts` / `Fire.ts` / `Cooking.ts` / `RestSpot.ts`: Interactable から
  「演出シーケンス」への再編。視覚・音の演出関数はそのまま活かす
- `ui/hints.ts`: 廃止

維持（無改修または軽微な配線変更のみ）:

`systems/GameState.ts`（資源状態 + 選択履歴の追加のみ）、`audio/`（AudioEngine・synths・
Reverb・footsteps）、`pano/`（PanoScene・SpotManager・StarMask・Snowfall・Gusts）、
`foreground/`（Breath・SitSequence・SitTimeline）、`core/`（Engine・PauseGate）、
`ui/`（Title・Credits・VolumeControl・DebugOverlay）。Spot / audioMix 設計は不変。

## 5. 写真と素材計画

- 新規スポット: 湖畔・森の小道の 2 枚。v2 の写真選定原則「自分では行けない場所へ連れて行く」を
  継承し、E2 方式の写真先行調査（Poly Haven / ambientCG の CC0 優先、無ければ Wikimedia の
  CC BY / CC BY-SA）で実在確認してから採用を確定する
- 予算改定（CLAUDE.md 更新事項）: 「3 枚合計 35MB 以下」→「1 枚 12MB 以下・スポット単位の
  遅延ロード（計 5 枚 = 現行 3 + 新規 2）」。初回ロードは campsite 1 枚のみで従来と変わらない
- 帰属表示は従来通り `ATTRIBUTION.md` + アプリ内クレジット

## 6. やらないこと

- マルチエンディング・フラグ蓄積による到達可否の分岐
- 実績・カウンター・進捗バー・回想リスト
- セーブ機能（一泊 15〜20 分・一気に過ごす前提。中断はタイトルからやり直し）
- BGM・タイプライター効果音の追加（音は環境音 + 行動音のまま）
- ノベルエンジン等の新規依存（Vanilla TS のまま。dependencies は three / simplex-noise / alea を維持）

## 7. テスト

- Vitest（StoryEngine / scenario）: 全ノード到達可能性・選択肢の行き先実在・合流検証
  （どの選択列でも幕に到達する）・表示条件の評価。GameState 既存テストは維持
- hints.ts のテスト 11 件は廃止に伴い削除
- Playwright: 選択肢クリックだけで一泊を完走（午後→夕→夜→朝→幕）、pageerror ゼロ、
  E キー押下が何も起こさないことの確認
